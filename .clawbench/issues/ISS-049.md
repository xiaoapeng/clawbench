## Description
In Codex resume mode (buildCodexResumeArgs), model_provider=minimax is unconditionally hardcoded when req.Model != "". This routes all model requests to MiniMax, causing wrong-model responses or provider errors for non-MiniMax models (e.g., OpenAI o3 via Codex).

## Impact
All Codex resume sessions with a model override will use the wrong provider. Users configuring non-MiniMax models will get incorrect AI responses or errors, with no clear error message indicating the cause.

## Suggestion
Make model_provider configurable via agent YAML, or omit it entirely and let Codex's own config determine the provider. At minimum, only set model_provider=minimax when the model name contains "minimax".
