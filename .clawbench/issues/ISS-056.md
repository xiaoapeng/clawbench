## Description
Path traversal via `clawbench_project` cookie — the project path read from the cookie is used directly in filesystem operations without validation. An attacker can set this cookie to `../../etc` or similar to point operations at arbitrary directories.

## Impact
Cross-directory access: an attacker can read, write, or delete files outside the intended project root by manipulating the cookie value.

## Suggestion
Validate the `clawbench_project` cookie value on every request: resolve it to an absolute path, clean it, and verify it is within the configured project root. Reject requests where the resolved path escapes the root.
