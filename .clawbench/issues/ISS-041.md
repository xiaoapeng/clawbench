## Description
`ServeFileMove` (line 374) uses `os.Rename` to move files but does not check whether the destination file already exists. On Unix systems, `os.Rename` atomically replaces the destination, silently destroying the target file. This is inconsistent with `ServeFileCopy` which explicitly checks for destination existence (line 410-413).

## Impact
A malicious or accidental move operation can silently overwrite any file within the WatchDir, resulting in data loss. This is a security vulnerability as it allows file destruction without explicit confirmation.

## Suggestion
Add an existence check before the move operation, mirroring the pattern in `ServeFileCopy`:
```go
if _, err := os.Stat(destAbsPath); err == nil {
    writeLocalizedErrorf(w, r, http.StatusConflict, "FileAlreadyExists")
    return
}
```
