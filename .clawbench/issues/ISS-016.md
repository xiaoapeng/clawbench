## Description
Tunnel connected event publishes `activeChannels` outside the mutex lock, reading a stale value. `activeChannels` can be concurrently modified by `handleDirectTCPIP` goroutines. This is a defined Go data race.

## Impact
`go test -race` would flag this. Published tunnel status events may contain inconsistent snapshots (e.g., clientCount=1 but activeChannels=0 when channels are already open), causing misleading UI.

## Suggestion
Capture both `connCount` and `activeChannels` inside the locked section, then publish outside the lock.
