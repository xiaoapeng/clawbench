## Description
RAG search endpoint `ServeRAGSearch()` does not call `requireProject()` to enforce project isolation. When the `project` field is empty in the request body, the SQL query omits the `WHERE project_path = ?` clause, returning results from all projects. The same issue affects `ServeRAGMessage` and `ServeRAGSession`.

## Impact
A user authenticated to one project can read the full text content of chat messages from any other project stored in the shared DuckDB instance, including potentially sensitive code, API keys, and confidential data.

## Suggestion
Call `requireProject(w, r)` in all RAG handlers and use the cookie-derived project path as the mandatory ProjectPath filter, ignoring any client-supplied project field.
