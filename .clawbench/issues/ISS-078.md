## Description
DeleteSession handler lacks project-ownership check — any authenticated user can delete chat sessions belonging to other projects. The handler processes the delete without verifying that the session belongs to the user's current project.

## Impact
Cross-project data deletion: a user can delete another project's chat history, causing data loss and potentially disrupting active sessions.

## Suggestion
Add a project-ownership check before processing the delete: verify the session's project matches the authenticated project context. Return 403 Forbidden on mismatch.
