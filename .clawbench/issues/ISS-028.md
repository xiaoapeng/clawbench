## Description
`StreamParser.receivedPartial*` flags are never reset during a stream. Once set to `true` (after seeing a `stream_event` message), these flags persist for the entire stream lifetime. If a CLI produces `stream_event` messages followed by verbose `assistant` messages (e.g., due to mode fallback or format change), ALL text and thinking content from subsequent `assistant` messages is silently dropped because the flags suppress them.

## Impact
Latent data loss bug — AI content silently dropped when stream format changes mid-stream, resulting in "AI returned no content" errors for the user.

## Suggestion
Reset `receivedPartial`, `receivedPartialThinking`, `receivedPartialToolUse` flags when a new `message_start` stream_event is received, since each `message_start` begins a fresh assistant turn.
