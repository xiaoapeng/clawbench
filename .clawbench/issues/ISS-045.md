## Description
SSE/DB data inconsistency when sendEvent drops events. When sendEvent returns false (context cancelled), AccumulateBlock is never called for that event, causing data loss in both SSE and DB. The finalized message may be incomplete.

## Impact
Data loss in DB persistence — AI responses can be partially recorded with no recovery path. The user sees a truncated message with no indication of data loss.

## Suggestion
Call AccumulateBlock BEFORE sendEvent, or ensure the current event is accumulated before finalizing:
```go
ai.AccumulateBlock(&blocks, event)
if !sendEvent(ctx, streamCh, event) {
    return finalizeStreamRun(...)
}
```
