## Description
CodexBackend missing `CLAWBENCH_SCHEDULED=1` environment variable — since it bypasses CLIBackend, the anti-recursion guard for scheduled tasks is not set. When a scheduled task triggers Codex, it may recursively invoke task creation commands without the guard.

## Impact
Anti-recursion bypass: scheduled tasks using the Codex backend can trigger infinite loops of task creation, each spawning new AI executions that create more tasks.

## Suggestion
Add `CLAWBENCH_SCHEDULED=1` to the environment variables in CodexBackend's command construction. Ensure this is set whenever the execution originates from a scheduled task, matching the behavior of CLIBackend-based backends.
