## Description
`isPortInRange` with empty `AllowedPorts` allows ALL ports, including privileged ports (<1024). When `ProxyConfig.AllowedPorts` is empty string, `isPortInRange` returns `true` for every port including system ports (22, 80, 443).

## Impact
A user who explicitly sets `allowed_ports:` (empty) in config.yaml would silently open all ports including privileged system ports, bypassing the intent of port-level access control.

## Suggestion
When `rangeStr` is empty, return `false` (deny-by-default) or at minimum exclude privileged ports (<1024).
