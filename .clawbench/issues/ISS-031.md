## Description
Anti-recursion check (`CLAWBENCH_SCHEDULED=1` env var) only protects the `create` subcommand. A scheduled AI execution can call `clawbench task update` to modify existing task prompts, cron schedules, or agent assignments — effectively creating new recursive behavior by modifying an existing task. This partially bypasses the anti-recursion protection.

## Impact
Partial anti-recursion bypass — scheduled AI agents can modify existing tasks for persistence or recursive execution.

## Suggestion
Add the same `CLAWBENCH_SCHEDULED` check to `runUpdate`:
```go
func runUpdate(args []string) int {
    if os.Getenv("CLAWBENCH_SCHEDULED") == "1" {
        return outputError("scheduled execution cannot update tasks")
    }
```
