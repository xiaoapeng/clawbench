## Description
`startGlobalPolling` interval is fixed at creation time and never adapts to SSE connectivity changes. `getInterval()` is called only once when `setInterval` runs. If SSE connects AFTER polling starts, the interval stays at 2s. If SSE disconnects after initially being connected, the interval stays at 60s, causing delayed detection of session completions.

## Impact
Either wasted resources (2s polling when SSE is connected) or delayed session completion detection (60s polling when SSE is disconnected), depending on the initial SSE state at polling start time.

## Suggestion
Use `setTimeout`-based recursive scheduling instead of `setInterval`, so `getInterval()` is re-evaluated on each tick.
