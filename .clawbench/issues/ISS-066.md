## Description
ClientID forgery allows cross-session file watch manipulation. The ClientID used to register file watch subscriptions is taken from the request without authentication or validation. An attacker can forge a ClientID belonging to another session to add/remove watches on their behalf.

## Impact
Cross-session interference: an attacker can inject unwanted file watch events into another user's SSE stream, or unsubscribe them from watches, causing missed notifications.

## Suggestion
Bind ClientID to the authenticated session (e.g., derive it from the session token or a server-side session ID). Reject ClientIDs that don't match the authenticated session. Alternatively, generate ClientID server-side and return it to the client.
