## Description
AutoResumeBackend resume request drops ThinkingEffort and ScheduledExecution fields. When constructing the resume request (e.g., after ExitPlanMode), these fields from the original request are not carried forward.

## Impact
Thinking effort configuration is lost on resume, causing the AI to use default effort instead of the user's selection. Scheduled execution metadata is lost, potentially breaking the anti-recursion guard and scheduled task tracking.

## Suggestion
Copy ThinkingEffort and ScheduledExecution (and any other relevant fields) from the original request into the resume request before executing it.
