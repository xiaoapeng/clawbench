## Description
When the Anthropic or OpenAI API returns a non-200 status, the full response body is read with `io.ReadAll` and included verbatim in the error string. Many API providers include the request details (including the API key or authorization header) in their error response bodies. This error string propagates to logs (via `slog.Warn`) and potentially to SSE events.

## Impact
API keys could be exposed in server logs and in HTTP responses to the frontend client, enabling credential theft. The attack surface extends to any OpenAI-compatible proxy configured via base_url.

## Suggestion
Truncate error body to 512 bytes; strip key-like patterns; log full body only at debug level and return sanitized message upstream.
