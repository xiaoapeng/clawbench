## Description
`useCrudList` uses module-level refs (`_items`, `_loaded`, `_showEditDialog`) that are shared across ALL calls regardless of `apiPrefix`. This means `useQuickSend` (apiPrefix: `/api/chat/quick-send`) and `useQuickCommands` (apiPrefix: `/api/terminal/quick-commands`) share the same reactive state. When one list is loaded, the other sees the same items. When one list's items are reordered, the other list's display is also affected.

## Impact
Real data corruption bug — quick-send and quick-commands lists crosstalk and show each other's data. Reorder operations on one list affect the other.

## Suggestion
Move the shared refs inside the `useCrudList()` function so each call gets its own refs, OR use a `Map<string, Ref>` keyed by `apiPrefix` to scope the state:
```typescript
const stores = new Map<string, ReturnType<typeof createStore>>()
function useCrudList<T, E>(apiPrefix: string, ...) {
  if (!stores.has(apiPrefix)) {
    stores.set(apiPrefix, createStore<T, E>(apiPrefix, ...))
  }
  return stores.get(apiPrefix)!
}
```
