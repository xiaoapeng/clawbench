## Description
TTS handler captures `http.Request` reference inside background goroutine for `T(r, ...)` localization calls after the HTTP handler has returned. After `TTSGenerate` returns, the underlying connection may be reused or closed by the server.

## Impact
Reading locale from stale request is unsafe and may cause panics or corrupted data.

## Suggestion
Resolve the locale string before starting the goroutine and capture the resolved string, not the request reference.
