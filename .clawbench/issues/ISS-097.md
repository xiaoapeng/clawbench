## Description
showCtx(entry=null) crashes on `entry.name` access — when `showCtx` is called with a null/undefined `entry` (e.g., from a right-click on empty space), accessing `entry.name` throws a TypeError and crashes the component.

## Impact
Runtime crash when right-clicking on empty space in the file manager. The UI becomes unresponsive until the page is reloaded.

## Suggestion
Add a null guard at the start of `showCtx`: return early or show a default context menu if `entry` is null/undefined. Alternatively, ensure the caller never passes null.
