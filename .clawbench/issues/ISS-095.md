## Description
stopPrimeTimer reference is undefined — `onBeforeUnmount` calls `clearTimeout(stopPrimeTimer)`, but `stopPrimeTimer` is never declared as a variable. This causes a runtime error every time the component is unmounted.

## Impact
Runtime error on component unmount. The prime timer is never cleared, potentially causing callbacks to fire after the component is destroyed, leading to "state update on unmounted component" warnings and memory leaks.

## Suggestion
Declare `stopPrimeTimer` as a ref or variable and assign it when `setTimeout` is called. Alternatively, if the prime timer is no longer needed, remove the `clearTimeout` call in `onBeforeUnmount`.
