## Description
serveProjectsCreate — `req.Name` is not sanitized for path traversal. A project create request with a name containing `../` can create a project directory at an arbitrary filesystem location.

## Impact
Arbitrary directory creation on the server. Could be used to plant files in sensitive locations or interfere with other projects' data.

## Suggestion
Sanitize `req.Name` to reject `..` components and path separators. Validate the resolved project path is within the configured projects root.
