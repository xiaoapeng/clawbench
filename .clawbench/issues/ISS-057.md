## Description
Upload filename not fully sanitized — only spaces are replaced, with no protection against `..` sequences or path separators (`/`, `\`). A malicious filename like `../../../etc/crontab` could write files outside the upload directory.

## Impact
Arbitrary file write on the server. An attacker uploading a file with a crafted name could overwrite critical system or application files.

## Suggestion
Apply comprehensive filename sanitization: strip path separators, reject `..` components, and optionally hash or truncate the name. Use `filepath.Base()` to extract just the filename portion before joining with the upload directory.
