## Description
EventSource connection does not pass `?token=` for WebView cookie-based auth. When cookies are not sent (cross-origin, Incognito, WebView restrictions), SSE connection fails with 401 but `onerror` cannot distinguish auth failure from network failure. Reconnect logic loops indefinitely on auth failures.

## Impact
UI appears stuck in reconnect loop when auth is invalid, never triggering re-auth flow. No graceful degradation for auth failures vs network failures.

## Suggestion
After `onerror`, attempt a lightweight authenticated fetch (e.g. `fetch('/api/ssh/info')`) to determine if the session is still valid. If 401, stop reconnecting and trigger re-auth flow.
