## Description
ServeFileRename — the new filename from the request body is not sanitized for path traversal. A rename request with `newName: "../../../etc/passwd"` could move a file outside the project root.

## Impact
Files can be moved to arbitrary locations on the server filesystem, enabling privilege escalation or data exfiltration.

## Suggestion
Sanitize the new filename using `filepath.Base()` and reject any input containing path separators or `..` components. Validate the resulting path stays within the project root after the rename.
