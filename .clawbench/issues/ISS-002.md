## Description
`requireProject` in SystemEventsSSE discards project path but EventBus is global — events from ALL projects broadcast to ALL clients. In multi-project scenarios, a client viewing Project A receives events from Project B.

## Impact
False `chatRunning` indicators, incorrect `runningSessions` tracking, and misleading UI state for multi-project users.

## Suggestion
Add project-scoping to EventBus by including `projectPath` in `SystemEvent.Payload` and filtering in the SSE handler or frontend `coreHandlers`.
