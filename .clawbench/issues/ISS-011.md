## Description
Cancelled execution does not update DB status to "cancelled" when context cancellation is detected after the event channel loop exits. If the CLI process exits normally around the same time the context is cancelled, `receivedTerminal` flag will be `true` and the function falls through to the success path.

## Impact
Execution will be marked "completed" even though it was cancelled. Race between cancellation and normal completion means CLI output may be truncated but still persisted as complete result.

## Suggestion
Check `ctx.Err()` first (before checking `receivedTerminal`) after the channel drain, or use `select` to prioritize cancellation detection.
