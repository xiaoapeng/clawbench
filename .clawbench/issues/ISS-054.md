## Description
MarkdownPreview.vue uses sanitize: false when rendering markdown, bypassing DOMPurify. The comment says "trusted file content" but AI agents can write files with arbitrary content including <script> tags. A user viewing such a file in rendered mode would execute the script.

## Impact
XSS via AI-generated markdown files. An AI agent writing a file with embedded <script> or <img onerror=...> tags would achieve code execution in the user's browser when the file is viewed in rendered mode.

## Suggestion
Remove sanitize: false and keep DOMPurify enabled. Add needed tags (math, etc.) to DOMPurify's allowlist instead of disabling sanitization entirely. The chat rendering path correctly uses DOMPurify; MarkdownPreview should follow the same pattern.
