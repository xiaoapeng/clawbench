## Description
`error` SSE event handler calls `disconnectStream()` AFTER `onLoadHistory()` starts, allowing stale SSE events to arrive during async reload. While the history reload is in-flight, the EventSource is still open and buffered/in-flight SSE events could mutate `streamingMsg.blocks`, overwriting freshly loaded DB state. Compare with the `done` handler which correctly calls `disconnectStream()` BEFORE `onLoadHistory()`.

## Impact
Race condition where SSE-driven state mutations overwrite freshly loaded DB state. Can cause UI inconsistencies during error recovery.

## Suggestion
Move `disconnectStream()` before `onLoadHistory()`, matching the pattern in the `done` handler.
