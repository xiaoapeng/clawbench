## Description
When `req.Resume` is true but `req.SessionID` is empty (session_capture event was missed or never emitted), `buildDeepSeekStreamArgs` produces neither `--resume` nor `--continue`. The `--continue` flag exists exactly for this scenario but is never used.

## Impact
DeepSeek resume silently degrades to a brand-new session. The user's conversation history is lost from the AI's context, and they may not notice.

## Suggestion
Add branch: `else if req.Resume { args = append(args, "--continue") }`
