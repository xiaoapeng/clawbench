## Description
CodexBackend completely bypasses CLIBackend abstraction — it duplicates process lifecycle management, output parsing, and error handling code instead of reusing the shared CLIBackend infrastructure. This creates a parallel implementation that must be maintained independently.

## Impact
Code duplication increases maintenance burden and the risk of divergent bug fixes. Changes to CLIBackend (e.g., security fixes, feature additions) must be manually replicated to CodexBackend, and missing replications cause silent regressions.

## Suggestion
Refactor CodexBackend to extend CLIBackend like other backends, providing only Codex-specific args and a LineParser. Move shared lifecycle code (process start/stop, context cancellation, output piping) into CLIBackend where it belongs.
