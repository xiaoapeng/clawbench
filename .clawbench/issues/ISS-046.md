## Description
CRLF header injection in ServeLocalFile — no filename sanitization in Content-Disposition header. The archive endpoint uses sanitizeArchiveName but ServeLocalFile does not. A file named with embedded \n characters would inject a second HTTP header.

## Impact
HTTP header injection allows response splitting attacks. An attacker with file creation access (e.g., via AI agent) could inject Set-Cookie headers or redirect responses.

## Suggestion
Filter \r and \n from fileName before setting Content-Disposition header. Reuse sanitizeArchiveName from file_archive.go and add explicit \r handling.
