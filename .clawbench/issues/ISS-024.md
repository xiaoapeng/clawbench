## Description
TTS handler goroutine silently drops events when SendTTSEvent returns false (channel full), including "result" events. Client may never receive completion and stays stuck forever.

## Impact
Frontend stuck in generating state when TTS event channel is full. No indication of failure.

## Suggestion
Log when events are dropped. For "result" events, use blocking send with timeout or fallback mechanism to ensure delivery.
