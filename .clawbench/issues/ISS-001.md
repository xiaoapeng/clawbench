## Description
Named SSE event listeners and `message` listener both registered in `useSystemEvents`, creating double-dispatch risk and format inconsistency. The `message` listener expects `{ type, payload }` wire format while named listeners parse `e.data` as just the `payload`. If the server ever sends an unnamed event, it will be dispatched with wrong structure.

## Impact
Latent inconsistency that could cause subtle double-dispatch bugs if server format changes. The `message` listener is effectively dead code but creates a format inconsistency.

## Suggestion
Remove the `message` listener (lines 225-233). All system events use named `event:` types, so it serves no purpose.
