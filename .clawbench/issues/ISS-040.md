## Description
`ServeFileDelete` (line 143-147) and `ServeFileBatchDelete` (line 206-210) ignore the error return from `os.RemoveAll` and `os.Remove`. The response always returns `ok: true` even if the file system operation failed (e.g., permission denied, file locked on Windows). In `ServeFileBatchDelete`, the `deleted` counter increments regardless of success, inflating the reported count.

## Impact
Clients receive a false success confirmation. The file remains on disk but the UI treats it as deleted. This can lead to confusion and data inconsistency, especially on Windows where files may be locked by running processes.

## Suggestion
Check the error return from `os.Remove`/`os.RemoveAll` and return an appropriate error response:
```go
if info.IsDir() {
    if err := os.RemoveAll(absPath); err != nil {
        model.WriteError(w, model.Internal(fmt.Errorf("delete failed: %w", err)))
        return
    }
} else {
    if err := os.Remove(absPath); err != nil {
        model.WriteError(w, model.Internal(fmt.Errorf("delete failed: %w", err)))
        return
    }
}
```
