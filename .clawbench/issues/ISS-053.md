## Description
All SSE event handlers do raw JSON.parse(e.data) with zero error handling. A single malformed JSON event from the backend throws an uncaught exception that breaks the stream, leaving the frontend in an inconsistent state (loading stuck true, timeout still ticking). Unlike the polling fallback which has try/catch, the SSE path has none.

## Impact
Stream crash with no recovery — the user must refresh the page. In the worst case, a malicious or corrupted SSE event could prevent all future chat interactions for that session.

## Suggestion
Wrap each JSON.parse(e.data) in try/catch. On failure, log the error and skip the event (don't kill the stream). This matches the robustness of the polling fallback path.
