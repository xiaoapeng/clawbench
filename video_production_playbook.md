# 推广视频制作流程经验文档

> 基于 ClawBench 推广视频实战总结，项目目录：`/home/xulongzhe/projects/推广视频`

---

## 一、流程总览

```
选题定位 → 脚本草稿 → PPT 制作 → 口播稿拆分 → TTS 语音生成 → 录屏素材 → 视频合成 → 输出
```

全流程核心原则：**每个章节独立生产、独立对齐、独立合成**，最后拼接成完整视频。

---

## 二、各环节详解

### 1. 选题定位与脚本规划

**产出：** `ppt/sources/script.md`

- 明确视频目标受众和核心信息（本例：面向开发者，核心是 ClawBench vs 竞品的差异化）
- 先写完整的脚本初稿，包含：
  - 每个章节的标题和时间预估
  - 口播内容（带格式标注，如加粗关键词）
  - 画面提示（括号标注，如 `（画面：打开文件浏览器 → 点进项目目录）`）
  - 语速标注（本例：约 280 字/分钟，偏快）
- 脚本结构按"钩子 → 痛点 → 差异化功能逐一展开 → 结尾总结"的逻辑编排

### 2. PPT 制作

**产出：** `ppt/svg_final/*.svg`、`ppt/png_final/*.png`、`ppt/exports/*.pptx`

目录结构：
```
ppt/
├── sources/       # 脚本源文件
│   └── script.md
├── images/        # 产品截图素材（.png）
├── svg_final/     # 每章一页 SVG（01_封面.svg ~ 13_结尾总结.svg）
├── png_final/     # SVG 渲染成 PNG（用于视频合成）
├── notes/         # 每章的口播提示卡（含要点、时长、弹性说明）
│   ├── 01_封面.md ~ 13_结尾总结.md
│   └── total.md   # 全部合并版
└── exports/       # 导出的 PPTX 文件
```

关键经验：
- **一章一 SVG**，文件名与章节编号对应（`01_封面.svg`、`02_开场钩子.svg`...）
- SVG → PNG 转换用于后续视频合成时的画面帧
- 产品截图放在 `images/` 下，SVG 中引用
- 每章同时维护一个 notes 文件，记录口播要点、时长预估和弹性空间
- 导出两版 PPTX：普通版和 SVG 嵌入版

### 3. 口播稿拆分

**产出：** `tts/*.txt`（每章一个 txt）、`script_tts.md`（完整口播版）

- 将脚本中的口播内容按 PPT 章节拆分成独立 txt 文件
- 拆分原则：
  - 每个文件对应一个 PPT 章节，编号对齐
  - 去掉画面提示和格式标注，只保留纯口播文字
  - 每个文件独立，可以单独送 TTS 生成语音
- `script_tts.md` 是完整的纯口播版（无画面提示），用于整体审校
- **注意：** tts/ 中的文字和 notes/ 中的略有不同——tts 版经过口语化调整，notes 版保留了过渡语和停顿提示

### 4. TTS 语音生成

**产出：** `tts/*.mp3`、`tts/tts_1.5x_*.aac`

- 使用微软 TTS 引擎生成语音
- 每个 txt 文件独立生成一个 mp3
- 1.5x 加速版：将 mp3 转为 aac 并加速（`atempo=1.5`），用于实际视频
  - 命名规则：`tts_1.5x_{章节名}.aac`
  - 某些章节有 v2 版本（如 `tts_1.5x_02_开场钩子_v2.aac`），说明有迭代
- 有些章节需要合并（如 `tts_1.5x_11_终端TTS_merged.aac`）

### 5. 录屏素材

**产出：** `recordings/*.mp4`

- 在 ClawBench 中实际操作录屏，每个功能点一个视频
- 录制内容：
  | 文件名 | 对应功能 |
  |--------|---------|
  | 定时任务.mp4 | 定时任务管理 |
  | 端口转发.mp4 | 端口映射 |
  | 交互式AI编辑.mp4 | AI 工具调用全程可见 |
  | 视频合成.mp4 | 视频制作流程本身 |
  | 文件管理器.mp4 | 文件浏览与管理 |
  | 终端界面.mp4 | Web 终端 |
  | Git历史.mp4 | Git 可视化 |
  | TTS朗读.mp4 | TTS 语音朗读 |
- 录屏素材在合成时叠加到对应章节

### 6. 视频合成

**产出：** `output/segments/*.mp4`、`output/full_video.mp4`、`output/mobile_full.mp4`

```
output/
├── segments/      # 每章独立视频片段（01_封面.mp4 ~ 13_结尾总结.mp4）
├── full_video.mp4 # 完整横版视频（~30MB）
├── mobile/        # 竖版章节片段（03~11，01/02/12/13 无竖版）
└── mobile_full.mp4 # 完整竖版视频（~400MB）
```

合成流程：
1. 每个章节独立合成：PPT 画面（PNG）+ 语音（1.5x AAC）+ 录屏素材
2. 13 个章节片段合成完整视频
3. 两个版本：
   - **横版**（`full_video.mp4`）：桌面端/YouTube
   - **竖版**（`mobile_full.mp4`）：手机端/短视频平台

## 四、关键经验

### 命名规范
- 统一用 `序号_章节名` 命名，如 `01_封面`、`04_文件工作流`
- 全链路保持一致：SVG、PNG、TXT、MP3、AAC、MP4 全部对齐同一编号
- 这样语音、画面、字幕天然对齐，不需要手动同步

### 章节独立原则
- 每个 PPT 页对应一个口播 txt、一段 TTS 语音、一个视频片段
- 好处：
  - 修改某章不影响其他章节
  - TTS 可以按章节独立重新生成
  - 视频合成可以逐章节迭代
  - 最终拼接顺序灵活调整

### TTS 加速
- 1.5x 加速后语速更紧凑，适合短视频节奏
- 保留原速 mp3 作为备份，加速版用 aac 格式节省体积
- 某些章节需要多次迭代 TTS（v2 版本）

### 双版本输出
- 横版：用于 YouTube/B站等桌面场景
- 竖版：用于抖音/视频号等手机场景
- 竖版不是所有章节都需要（01 封面、02 开场钩子、12 部署、13 结尾无竖版）

### 口播稿 vs 脚本的区别
- `script.md`：带画面提示、格式标注、时间标记的完整脚本
- `script_tts.md`：纯口播文字，无画面提示
- `tts/*.txt`：每章独立的口播文本，经过口语化调整
- `ppt/notes/*.md`：带要点、时长、弹性说明的提示卡

### 工具链
- PPT 制作：SVG（可用 AI 辅助生成）→ 渲染 PNG
- TTS：微软 TTS
- 音频加速：ffmpeg atempo
- 视频合成：ffmpeg
- 全程可在 ClawBench 中完成（自己吃自己的狗粮）
